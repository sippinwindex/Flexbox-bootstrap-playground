Getting Started with CSS

Beginner

CSS (Cascading Style Sheets) controls the visual presentation of HTML elements. It defines colors, layouts, fonts, spacing, animations, and much more.

What You'll Master
  • CSS Fundamentals: Syntax, selectors, specificity, and the cascade
  • Layout Systems: Box model, Flexbox, CSS Grid, and positioning
  • Modern CSS: Custom properties, responsive design, and animations
  • Best Practices: Performance, accessibility, and maintainable code

Three Ways to Add CSS

Styled with inline CSS

Use Case: Quick testing only. Avoid in production.

    color: #0c5460;

CSS Selectors

Beginner

Selectors determine which HTML elements your CSS styles will be applied to. Understanding selectors is crucial for effective CSS.

Basic Selectors

Element, Class, and ID Selectors

Element Selector: All <p> elements

Class Selector: .highlight
ID Selector: #header
CSS 
/* Element Selector - targets all paragraph elements */
p {
    color: green;
    font-size: 16px;
}

/* Class Selector - targets elements with class="highlight" */
.highlight {
    background-color: yellow;
    padding: 10px;
    border-radius: 4px;
}

/* ID Selector - targets element with id="header" */
#header {
    background-color: #333;
    color: white;
    text-align: center;
    padding: 10px;
}

    background-color: #f8f9fa;

p:first-child {
    margin-top: 0;

    margin-bottom: 0;
}
Selector Best Practices
  • Use classes for reusable styles
  • Use IDs sparingly (only for unique elements)
  • Avoid overly specific selectors
  • Use semantic class names (e.g., .warning instead of .red)
  • Keep specificity as low as possible

Important

Every HTML element is essentially a box. The CSS box model describes how these boxes are rendered, including content, padding, borders, and margins.

CSS 
.box-model-demo {
    /* Content dimensions */
    width: 200px;
    height: 100px;
    
    /* Padding - space between content and border */
    padding: 20px;
    
    /* Border - line around the padding and content */
    border: 5px solid #333;
    
    /* Margin - space outside the border */
    margin: 15px;
    
    /* Background only covers content + padding */
    background-color: #e3f2fd;
}

/* Total element width calculation:
   content-box: 200px + 40px (padding) + 10px (border) = 250px
   border-box: 200px exactly (padding and border included)
*/

Box-sizing Property

Content-box vs Border-box
content-box (default)
Total: 260px wide
Width = 200px + 40px padding + 20px border = 260px
border-box (recommended)
Total: 200px wide
Width = 200px exactly (padding and border included)
CSS 
/* Content-box (default behavior) */
.content-box {
    box-sizing: content-box;
    width: 200px;
    padding: 20px;
    border: 10px solid blue;
    /* Total width = 200px + 40px padding + 20px border = 260px */
}

/* Border-box (recommended) */
.border-box {
    box-sizing: border-box;
    width: 200px;
    padding: 20px;
    border: 10px solid blue;
    /* Total width = 200px exactly */
}

/* Universal border-box (best practice) */
*, *::before, *::after {
    box-sizing: border-box;
}

/* Alternative universal approach */
html {
    box-sizing: border-box;
}

*, *::before, *::after {
    box-sizing: inherit;
}
Box Model Best Practice

Always use box-sizing: border-box for predictable layouts. Apply it universally with the CSS reset above to make width and height calculations much more intuitive!

Accessibility Note

Always ensure sufficient color contrast for text readability. Use tools like WebAIM's contrast checker to verify your color combinations meet WCAG guidelines (4.5:1 ratio for normal text, 3:1 for large text).

Layout & Positioning

Intermediate

Control how elements are positioned and laid out on the page using various CSS properties.

Position Property Values
Static positioning (normal flow)
Relative positioning (offset from normal)
Absolute positioning
Fixed positioning
CSS 
/* Static positioning (default) */
.static {
    position: static;
    /* Elements flow in normal document order */
}

/* Relative positioning */
.relative {
    position: relative;
    top: 10px;    /* Move down 10px from normal position */
    left: 20px;   /* Move right 20px from normal position */
    /* Original space is preserved */
}

/* Absolute positioning */
.absolute {
    position: absolute;
    top: 0;       /* Distance from nearest positioned parent */
    right: 0;     /* or from viewport if no positioned parent */
    /* Removed from normal document flow */
}

/* Fixed positioning */
.fixed {
    position: fixed;
    bottom: 20px; /* Always 20px from bottom of viewport */
    right: 20px;  /* Always 20px from right of viewport */
    /* Stays in place when scrolling */
}

/* Sticky positioning */
.sticky {
    position: sticky;
    top: 0;       /* Sticks to top when scrolling */
    /* Toggles between relative and fixed */
}

Block, Inline, and Modern Display Values
Block Elements (full width)

CSS 
/* Block elements (default for div, p, h1-h6, etc.) */
.block {
    display: block;
    width: 100%;           /* Takes full width available */
    margin-bottom: 10px;   /* Can set width and height */
    /* Starts on new line, forces next element to new line */
}

/* Inline elements (default for span, a, strong, etc.) */
.inline {
    display: inline;
    /* Width and height have no effect */
    /* Flows with text, doesn't break lines */
    /* Only horizontal margins/padding work */
}

/* Inline-block (best of both worlds) */
.inline-block {
    display: inline-block;
    width: 100px;         /* Can set width and height */
    height: 50px;         /* Respects vertical margins/padding */
    /* Flows inline but behaves like block for sizing */
}

/* Modern layout methods */
.flex-container {
    display: flex;        /* Creates flex container */
    /* Children become flex items */
}

.grid-container {
    display: grid;        /* Creates grid container */
    grid-template-columns: repeat(3, 1fr);
}

/* Hide elements */
.hidden {
    display: none;        /* Completely removes from layout */
}

.invisible {
    visibility: hidden;   /* Hides but keeps space */
}

Flexbox Layout

Modern

Flexbox provides an efficient way to arrange, distribute, and align elements in a container, even when their size is unknown or dynamic.

When to Use Flexbox
  • One-dimensional layouts (row or column)
  • Centering content vertically and horizontally
  • Distributing space between items
  • Creating flexible navigation bars

Flex Container Properties

Flex Direction and Justification

Item 1
Item 2
Item 3
justify-content: center
Centered

CSS 
.flex-container {
    display: flex;
    
    /* Main axis alignment (horizontal by default) */
    justify-content: flex-start;    /* Default - items at start */
    justify-content: center;        /* Center items */
    justify-content: flex-end;      /* Items at end */
    justify-content: space-between; /* Space between items */
    justify-content: space-around;  /* Space around items */
    justify-content: space-evenly;  /* Equal space between/around */
    
    /* Cross axis alignment (vertical by default) */
    align-items: stretch;      /* Default - stretch to fill */
    align-items: flex-start;   /* Align to top */
    align-items: center;       /* Vertical center */
    align-items: flex-end;     /* Align to bottom */
    align-items: baseline;     /* Align text baselines */
    
    /* Direction of flex items */
    flex-direction: row;         /* Default - left to right */
    flex-direction: row-reverse; /* Right to left */
    flex-direction: column;      /* Top to bottom */
    flex-direction: column-reverse; /* Bottom to top */
    
    /* Wrapping behavior */
    flex-wrap: nowrap;    /* Default - single line */
    flex-wrap: wrap;      /* Allow wrapping */
    flex-wrap: wrap-reverse; /* Wrap in reverse order */
    
    /* Shorthand for direction and wrap */
    flex-flow: row wrap;  /* direction wrap */
    
    /* Gap between items (modern browsers) */
    gap: 1rem;            /* Space between all items */
    row-gap: 1rem;        /* Space between rows */
    column-gap: 0.5rem;   /* Space between columns */
}

CSS 
.flex-item {
    /* How much item should grow (default: 0) */
    flex-grow: 1;     /* Takes available space */
    flex-grow: 2;     /* Takes twice as much space as flex-grow: 1 */
    
    /* How much item should shrink when space is limited (default: 1) */
    flex-shrink: 0;   /* Don't shrink */
    flex-shrink: 2;   /* Shrink twice as fast as default */
    
    /* Initial size before growing/shrinking (default: auto) */
    flex-basis: auto;    /* Use content size */
    flex-basis: 0;       /* Start from 0 size */
    flex-basis: 200px;   /* Start from 200px */
    flex-basis: 50%;     /* Start from 50% of container */
    
    /* Shorthand: flex-grow flex-shrink flex-basis */
    flex: 1;           /* Same as flex: 1 1 0% */
    flex: 1 0 auto;    /* Grow but don't shrink, auto basis */
    flex: 0 0 200px;   /* Fixed 200px width */
    flex: none;        /* Same as flex: 0 0 auto */
    
    /* Override container's align-items for this item */
    align-self: auto;      /* Use container's align-items */
    align-self: stretch;   /* Stretch to fill cross axis */
    align-self: center;    /* Center on cross axis */
    align-self: flex-start; /* Align to start of cross axis */
    align-self: flex-end;   /* Align to end of cross axis */
    
    /* Change visual order (doesn't affect screen readers) */
    order: 0;         /* Default order */
    order: 1;         /* Appears after order: 0 items */
    order: -1;        /* Appears before order: 0 items */
}

CSS Grid Layout

Modern

CSS Grid is a powerful two-dimensional layout system that allows you to create complex layouts with rows and columns.

When to Use CSS Grid
  • Two-dimensional layouts (rows and columns)
  • Complex page layouts with multiple areas
  • Card-based designs and image galleries
  • When you need precise control over item placement

Grid Container Properties

Basic Grid Layout
1
2
3
4
5
6

This grid uses: display: grid; grid-template-columns: repeat(3, 1fr); gap: 10px;

CSS 
.grid-container {
    display: grid;
    
    /* Define columns */
    grid-template-columns: 1fr 2fr 1fr;     /* Three columns: small, large, small */
    grid-template-columns: repeat(3, 1fr);   /* Three equal columns */
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); /* Responsive columns */
    grid-template-columns: 200px 1fr 100px; /* Mixed units */
    
    /* Define rows */
    grid-template-rows: 100px auto 50px;    /* Three rows with specific heights */
    grid-template-rows: repeat(3, 1fr);     /* Three equal rows */
    
    /* Gap between grid items */
    gap: 20px;              /* Gap between rows and columns */
    row-gap: 20px;          /* Gap between rows only */
    column-gap: 10px;       /* Gap between columns only */
    
    /* Alignment of grid items */
    justify-items: stretch; /* Default - items fill cell width */
    justify-items: start;   /* Items align to start of cell */
    justify-items: center;  /* Items center in cell */
    justify-items: end;     /* Items align to end of cell */
    
    align-items: stretch;   /* Default - items fill cell height */
    align-items: start;     /* Items align to top of cell */
    align-items: center;    /* Items center vertically in cell */
    align-items: end;       /* Items align to bottom of cell */
    
    /* Alignment of entire grid within container */
    justify-content: start; /* Grid aligns to start of container */
    justify-content: center; /* Grid centers in container */
    justify-content: end;   /* Grid aligns to end of container */
    
    align-content: start;   /* Grid aligns to top of container */
    align-content: center;  /* Grid centers vertically in container */
    align-content: end;     /* Grid aligns to bottom of container */
}

        display: none;

Responsive Design Best Practices
  • Mobile First: Start with mobile styles, then enhance for larger screens
  • Flexible Units: Use relative units (%, em, rem, vw, vh) instead of fixed pixels
  • Flexible Images: Use max-width: 100% and height: auto
  • Touch-Friendly: Make interactive elements at least 44px for touch targets
  • Test on Real Devices: Browser dev tools are helpful, but test on actual devices too

Responsive Units

CSS Units for Responsive Design
Relative Units
  • rem: Relative to root font size
  • em: Relative to parent font size
  • %: Relative to parent element
  • vw/vh: Viewport width/height
Fixed Units
  • px: Pixels (use sparingly)
  • pt: Points (mainly for print)
  • in/cm: Physical units
CSS 
/* Responsive units examples */
.responsive-typography {
    /* rem - relative to root element font size (usually 16px) */
    font-size: 1.5rem;    /* 24px if root is 16px */
    margin: 2rem;         /* 32px if root is 16px */
    
    /* em - relative to parent element font size */
    padding: 1em;         /* Same as current font size */
    
    /* Percentage - relative to parent container */
    width: 50%;           /* Half of parent width */
    height: 100%;         /* Full parent height */
    
    /* Viewport units */
    width: 100vw;         /* Full viewport width */
    height: 100vh;        /* Full viewport height */
    font-size: 4vw;       /* 4% of viewport width */
    
    /* Minimum and maximum with clamp() */
    font-size: clamp(1rem, 4vw, 2rem); /* Min 1rem, ideal 4vw, max 2rem */
    
    /* calc() for complex calculations */
    width: calc(100% - 2rem);  /* Full width minus 2rem */
    height: calc(100vh - 4rem); /* Full viewport height minus 4rem */
}

/* Flexible image styling */
.responsive-image {
    max-width: 100%;      /* Never exceed container width */
    height: auto;         /* Maintain aspect ratio */
    display: block;       /* Prevent inline spacing issues */

Animation Performance Tips
  • Animate transform and opacity for best performance
  • Use will-change property to hint browser optimization
  • Avoid animating layout properties (width, height, position)
  • Use CSS animations instead of JavaScript for simple effects
  • Test animations on lower-end devices

CSS Functions

Mathematical CSS
Responsive Title

Width: calc(100% - 2rem)
Font size: clamp(1rem, 4vw, 1.5rem)

CSS 
.advanced-functions {
    /* calc() - Mathematical calculations */
    width: calc(100% - 40px);
    height: calc(100vh - 4rem);
    margin: calc(2rem + 10px);
    
    /* clamp() - Responsive values with min/max */
    font-size: clamp(1rem, 2.5vw, 2rem); /* min, preferred, max */
    padding: clamp(0.5rem, 3vw, 2rem);
    
    /* min() and max() - Choose smaller/larger value */
    width: min(90%, 1200px);    /* Whichever is smaller */
    height: max(300px, 50vh);   /* Whichever is larger */
    
    /* Color functions */
    background: hsl(200, 50%, 50%);
    border-color: hsl(200, 50%, 30%); /* Darker version */
    
    /* Modern color functions (limited browser support) */
    background: color-mix(in srgb, blue 30%, red);
    background: color-contrast(white vs black, blue, red);
    
    /* attr() - Use HTML attribute values */
    content: attr(data-label);
    width: attr(data-width px, 100px); /* With fallback */
    
    /* CSS Grid functions */
    grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
    grid-template-columns: fit-content(300px) 1fr fit-content(200px);
    
    /* Transform functions */
    transform: rotate(45deg) scale(1.2) translateX(20px);
    transform: matrix(1, 0, 0, 1, 0, 0); /* Advanced 2D transforms */
    
    /* Filter functions */
    filter: blur(5px) brightness(1.2) contrast(1.1);
    backdrop-filter: blur(10px) saturate(150%);
}

Container Queries

Element-Based Responsive Design
Container Queries

Container queries are a newer CSS feature that allows elements to respond to their container's size rather than the viewport size. Support is still limited to modern browsers.

CSS 
/* Container query setup */
.card-container {
    container-type: inline-size; /* Enable container queries */
    container-name: card;         /* Optional: name the container */
}

/* Alternative shorthand */
.card-container {
    container: card / inline-size;
}

/* Container queries - respond to container size, not viewport */
@container card (min-width: 300px) {
    .card {
        display: flex;
        flex-direction: row;
        gap: 1rem;
    }
    
    .card-image {
        flex: 0 0 120px;
    }
}

@container card (max-width: 299px) {
    .card {
        display: flex;
        flex-direction: column;
    }
    
    .card-image {
        width: 100%;
        height: 200px;
    }
}

/* Container query without explicit name */
.sidebar {
    container-type: inline-size;
}

@container (min-width: 250px) {
    .sidebar-widget {
        padding: 2rem;
        font-size: 1.1rem;
    }
}

@container (max-width: 249px) {
    .sidebar-widget {
        padding: 1rem;
        font-size: 0.9rem;
    }
}

/* Container query units */
.responsive-text {
    /* cqw = 1% of container's width */
    /* cqh = 1% of container's height */
    /* cqi = 1% of container's inline size */
    /* cqb = 1% of container's block size */
    /* cqmin = smaller of cqi and cqb */
    /* cqmax = larger of cqi and cqb */
    
    font-size: 4cqw; /* 4% of container width */
    padding: 2cqi;   /* 2% of container inline size */
}
