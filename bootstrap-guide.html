Getting Started with Bootstrap

Beginner

Bootstrap is a powerful, feature-packed CSS framework that provides pre-built components and utilities for rapid web development with a mobile-first approach.

Why Choose Bootstrap?
  • Responsive Grid System: Flexbox-based 12-column grid
  • Pre-designed Components: Buttons, modals, navigation, cards, and more
  • Utility Classes: Spacing, colors, display, positioning utilities
  • Cross-browser Compatibility: Works consistently across modern browsers
  • Active Community: Extensive documentation and community support
  • Customizable: Sass variables and custom builds

Installation Methods

CDN (Recommended for Learning)
<!-- CSS -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">

<!-- JavaScript Bundle -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
Pros: Quick setup, automatic updates, CDN performance benefits.
Download
<!-- Local files -->
<link rel="stylesheet" href="css/bootstrap.min.css">
<script src="js/bootstrap.bundle.min.js"></script>
Pros: Offline development, version control, custom builds.
Package Manager
# npm
npm install bootstrap

# Yarn
yarn add bootstrap

# pnpm
pnpm add bootstrap
Pros: Dependency management, build tool integration, customization.

Bootstrap Grid System

Essential

Bootstrap's grid system uses containers, rows, and columns to layout and align content. It's built with flexbox and is fully responsive.

Grid Basics
  • Container: Wraps content and provides proper alignment
  • Rows: Horizontal groups of columns
  • Columns: Content placement within rows
  • 12-Column System: Columns add up to 12 per row
  • Gutters: Padding between columns

  • Always wrap columns in rows

Customization

Advanced

Learn how to customize Bootstrap to match your brand and design requirements.

CSS Custom Properties

/* Override Bootstrap's CSS custom properties */
:root {
    --bs-primary: #custom-color;
    --bs-secondary: #another-color;
    --bs-font-sans-serif: 'Your Custom Font', system-ui, sans-serif;
    --bs-border-radius: 0.5rem;
    --bs-box-shadow: 0 0.5rem 1rem rgba(0, 0, 0, 0.15);
}

/* Custom component styling */
.btn-custom {
    --bs-btn-color: #fff;
    --bs-btn-bg: #6f42c1;
    --bs-btn-border-color: #6f42c1;
    --bs-btn-hover-bg: #5a359a;
    --bs-btn-hover-border-color: #5a359a;
}

Custom Utility Classes

// Add custom utilities to the utilities map
$utilities: map-merge(
  $utilities,
  (
    "cursor": (
      property: cursor,
      class: cursor,
      responsive: true,
      values: auto grab pointer,

    "user-select": (
      property: user-select,

  )

// This generates classes like:

Customization Best Practices
  • Use CSS custom properties for simple theme changes
  • Use Sass for more complex customizations and builds
  • Never modify Bootstrap source files directly
  • Create a separate custom.scss file for your modifications
  • Test customizations across different breakpoints

Best Practices

Expert

Follow these best practices to build maintainable, accessible, and performant Bootstrap applications.

Performance
  • Use the CDN for faster loading
  • Only include components you need
  • Minimize custom CSS overrides
  • Optimize images and use responsive images
  • Use utility classes instead of custom CSS when possible
Accessibility
  • Use semantic HTML elements
  • Include proper ARIA attributes
  • Ensure sufficient color contrast
  • Test with keyboard navigation
  • Use screen reader testing tools
Responsive Design
  • Design mobile-first
  • Test on multiple screen sizes
  • Use appropriate breakpoints
  • Consider touch-friendly interfaces
  • Optimize for different orientations
Code Quality
  • Follow consistent naming conventions
  • Keep custom CSS organized
  • Document complex customizations
  • Use version control for your changes
  • Regular code reviews and testing

Common Mistakes to Avoid

  • Ignore mobile responsiveness

  • Use inline styles excessively
