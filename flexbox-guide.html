Introduction to Flexbox

Essential

CSS Flexbox (Flexible Box Layout) is a one-dimensional layout method that makes it easy to align and distribute space among items in a container, even when their size is unknown or dynamic.

Why Flexbox?
  • Flexible: Items grow and shrink to fit available space
  • Alignment: Easy vertical and horizontal centering
  • Direction: Lay out items in any direction
  • Order: Reorder items without changing HTML
  • Responsive: Perfect for responsive designs

CSS 
/* Create a flex container */
.flex-container {
    display: flex;           /* Creates flex formatting context */
    gap: 0.5rem;            /* Space between items */
    padding: 1rem;
    border: 3px solid #007bff;
    border-radius: 8px;
}

/* Flex items (children) automatically participate */
.flex-item {
    flex: 1;                /* Equal width distribution */
    padding: 1rem;
    text-align: center;
    border-radius: 4px;
}

/* Individual item styling */
.item-1 { background: #007bff; color: white; }
.item-2 { background: #28a745; color: white; }
.item-3 { background: #ffc107; color: black; }

    flex-direction: column;  /* Mobile: stack vertically */
    gap: 1rem;

    display: flex;

Flexbox Best Practices

Expert
Do's
  • Use Flexbox for one-dimensional layouts
  • Use gap instead of margins for spacing
  • Use flex: 1 for equal distribution
  • Start with mobile-first responsive design
  • Combine with CSS Grid for complex layouts
  • Test with varying content lengths
Don'ts
  • Don't use Flexbox for complex 2D layouts
  • Don't rely on order for content sequence
  • Don't forget about IE11 if supporting it
  • Don't use percentage widths with flex-basis unnecessarily
  • Don't forget flex items create stacking contexts
  • Don't use margins for spacing between flex items
Performance
  • Flexbox is generally more performant than floats
  • Avoid frequent changes to flex properties during animations
  • Use will-change for animated flex items
  • Consider content-visibility for large containers
Accessibility
  • Visual order should match logical order
  • Use order carefully with screen readers
  • Ensure sufficient focus indicators
  • Test with keyboard navigation
  • Use semantic HTML within flex containers
