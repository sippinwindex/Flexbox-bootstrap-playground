Introduction to Flexbox

Essential

CSS Flexbox (Flexible Box Layout) is a one-dimensional layout method that makes it easy to align and distribute space among items in a container, even when their size is unknown or dynamic.

Why Flexbox?
  • Flexible: Items grow and shrink to fit available space
  • Alignment: Easy vertical and horizontal centering
  • Direction: Lay out items in any direction
  • Order: Reorder items without changing HTML
  • Responsive: Perfect for responsive designs

Before Flexbox vs. After Flexbox

‚ùå Old Way (Float/Position)
/* Complex centering */
.center {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}

/* Clearfix for floats */
.clearfix::after {
    content: "";
    display: table;
    clear: both;
}
‚úÖ Flexbox Way
/* Simple centering */
.flex-center {
    display: flex;
    justify-content: center;
    align-items: center;
}

/* That's it! */

Advanced

3. flex-basis

.item {
    flex-basis: auto;    /* default: use width/height */
    flex-basis: 200px;   /* initial size before growing/shrinking */
    flex-basis: 20%;     /* percentage of container */
    flex-basis: 0;       /* ignore intrinsic size */
}

4. flex Shorthand

.item {
    flex: 1;              /* flex-grow: 1, flex-shrink: 1, flex-basis: 0% */
    flex: 0 0 200px;      /* don't grow/shrink, 200px wide */
    flex: 1 1 auto;       /* grow and shrink, auto basis */
    flex: none;           /* same as 0 0 auto */
}
Common Values
  • flex: 1 - Item takes up available space
  • flex: 0 0 auto - Item keeps its natural size
  • flex: 1 1 0% - All items have equal width

    align-self: stretch;    /* stretch to fill */

Advanced

Responsive Direction

.responsive-flex {
    display: flex;
    flex-direction: column;  /* mobile first: stack vertically */
}

@media (min-width: 768px) {
    .responsive-flex {
        flex-direction: row;  /* tablet+: horizontal layout */

    gap: 1rem;

Flexbox Best Practices

Expert
Do's
  • Use Flexbox for one-dimensional layouts (row or column)
  • Combine with CSS Grid for complex layouts
  • Use gap instead of margins for spacing
  • Use flex: 1 for equal distribution
  • Consider accessibility when reordering with order
  • Test on different screen sizes and content lengths
Don'ts
  • Don't use Flexbox for complex 2D layouts (use CSS Grid)
  • Don't rely on order for meaningful content sequence
  • Don't forget about IE11 if you need to support it
  • Don't use percentage widths with flex-basis unnecessarily
  • Don't forget that flex items create new stacking contexts

Performance Tips

Optimization
  • Flexbox is generally more performant than float-based layouts
  • Avoid frequent changes to flex properties during animations
  • Use will-change property for animated flex items
  • Consider content-visibility for large flex containers

Common Issues & Solutions

Debugging

Issue 1: Items Not Taking Full Height

‚ùå Problem
.container {
    display: flex;
    height: 200px;
}

.item {
    /* Items don't fill height */
    background: blue;
}
‚úÖ Solution
.container {
    display: flex;
    height: 200px;
    align-items: stretch; /* default, ensures full height */
}

.item {
    /* Now items fill container height */
    background: blue;
}

Issue 2: Items Overflowing Container

‚ùå Problem
.container {
    display: flex;
    width: 300px;
}

.item {
    min-width: 150px; /* Items won't shrink below 150px */
}
‚úÖ Solution
.container {
    display: flex;
    flex-wrap: wrap; /* Allow wrapping */
    width: 300px;
}

.item {
    flex: 1 1 150px; /* flex-basis instead of min-width */
}

Issue 3: Unexpected Spacing

‚ùå Problem
.container {
    display: flex;
    justify-content: space-between;
}

/* Only one item - unexpected positioning */
‚úÖ Solution
.container {
    display: flex;
    /* Use appropriate justify-content */
    justify-content: flex-start; /* or center */
}

/* Or use margin-left: auto on last item */

Issue 4: Flex Items Not Growing

‚ùå Problem
.item {
    flex-grow: 1;
    width: 200px; /* Fixed width prevents growing */
}
‚úÖ Solution
.item {
    flex: 1; /* Shorthand that sets flex-basis: 0% */
    /* Or explicitly: */
    flex-grow: 1;
    flex-basis: 0%;
}
Debugging Tips
  • Use browser dev tools to visualize flex containers
  • Add temporary borders to see element boundaries
  • Check for conflicting CSS properties (width, min-width, etc.)
  • Verify flex container has defined height for vertical alignment
  • Use Firefox Flexbox Inspector for detailed analysis
